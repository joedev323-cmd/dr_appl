it is a doctors application program built in spring boot 
it is built to simulate or rather explore how a cpu uses critical section in its process

here the patient and the doctor are the tasks to be performed 
and the room acts like the cpu's critical section

at any given time when the room is occupied(just like a toilet) no none else can book 
the specific room since it is in use

there is also the aspect of which two users can be present in the room at a given instance

there's who can book and how to prevent double booking at a given time for the 
given room.

the aim of this project is to reduce the time or cut the time and frustrations that
patients experience when going to book an appointment

eg long queues, calls not going through  among other frustrations.
